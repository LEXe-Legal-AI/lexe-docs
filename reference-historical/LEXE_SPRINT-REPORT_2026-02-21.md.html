Routing Architecture — Before vs After

BEFORE — Double Classifier
1. classify_complexity() → LLM call #1 (~2.5s)
2. run_intent_detector() → LLM call #2 (~1.5s)
3. Pipeline re-detects intent internally
Total: 3-4 LLM calls, 4-5s routing overhead
AFTER — Unified Flow
1. Fast-path: word_count < 8 & no legal hints → toolloop (0ms)
2. run_intent_detector() → single LLM call (~1.8s)
3. IntentResult passed to pipeline (skip re-detection)
Total: 1 LLM call, <2s routing overhead

Self-Correction Verifier Loop

verify_with_self_correction() — Max 2 attempts, 15s timeout
Attempt 1: run_verifier() → PASS = VERIFIED | FAIL → retry_with_reflection()
Attempt 2: run_verifier() → PASS = VERIFIED | FAIL = CAUTION + disclaimer
New model: VerificationResult (confidence_level: VERIFIED | CAUTION | LOW_CONFIDENCE)
